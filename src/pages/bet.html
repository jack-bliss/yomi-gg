%%user_info%% 
GET /profile/?page=1

Paginated, always thinks there are no more profiles left regardless of how many there are, page size is 10.

POST /auth/sign-up

Form params username, password, email. Password currently sent in plain text and hashed on server.

POST /auth/log-in

For params email, password. Password sent in plain text.

POST /auth/email-exists?email=blah@me.com

Just returns true or false in all lowercase if the email exists/doesn't.

POST /auth/username-exists?username=CoolGuy420

Same as above

POST /bet/match

Form params set_id, prediction, wager, everything else is read from session/server. have to have a valid session.

GET /bet/match

One big list of every bet the user has made on matches. Have to be logged in.

GET /bet/match/:set_id

Returns a breakdown of the bets made on a set with the matching id (what you posted to /bet/match). Has a total field, and a field for each prediction (again, from /bet/match). You have to be an admin for this.
